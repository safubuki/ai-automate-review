# 障害対応手順

## 目的

障害が発生した際に、迅速かつ正確に対応するための手順を記載します。

## 手順

### 1. 障害検知

ユーザから連絡があったり、監視アラートが鳴ったりしたら対応を開始してください。

### 2. 一次対応

とりあえずサーバにsshしてログを確認します。
その後、アプリケーションを再起動してみてください。
だいたいこれで直ると思います。

### 3. エスカレーション

対応が難しい場合は、チームリーダーに連絡します。

### 4. 復旧後

復旧したら、ユーザに「直りました」と連絡してください。
スクショは適当でいいと思います。

## 注意点

- ユーザという言葉は使わない方がいいです（用語集参照）
- スクショは最新にしてください
