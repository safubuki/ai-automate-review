# 障害対応手順

## 目的

障害が発生した際に、迅速かつ正確に対応するための手順を記載します。

## 前提条件

- 障害対応権限を持つアカウントでログインしていること
- 監視ツール（例: Datadog, Prometheus）にアクセスできること

## 手順

### 1. 障害検知

ユーザから連絡があったり、監視アラートが鳴ったりしたら対応を開始してください。

### 2. 一次対応

1. サーバにSSHでログインします。
2. アプリケーションログを確認してください。
3. 必要に応じてアプリケーションを再起動してみてください。

### 3. エスカレーション

対応が難しい場合は、チームリーダーに連絡します。

### 4. 復旧後

復旧したら、ユーザに「直りました」と連絡してください。

- ユーザという言葉は使わない方がいいです（用語集参照）
- スクショは最新にしてください
