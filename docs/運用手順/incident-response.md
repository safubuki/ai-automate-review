# 障害対応手順

## 目的

障害が発生した際に、迅速かつ正確に対応するための手順を記載します。

## 前提条件

- 障害対応権限を持つアカウントでログインしていること
- 監視ダッシュボードおよびログ基盤にアクセスできること
- インシデント通知用のチャットチャンネルに参加していること

## 影響範囲の確認

対応を開始する前に、以下を確認してください。

- 影響を受けているサービス・機能
- 影響を受けているユーザーの範囲
- 発生時刻と継続時間

## 手順

### 1. 障害検知

ユーザーからの連絡、または監視アラートを受信したら対応を開始します。

### 2. 一次対応

1. サーバーにSSH接続します。
2. アプリケーションログおよびシステムログを確認します。
3. ログから原因が特定できる場合は該当する復旧手順を実施します。
4. 原因が特定できない場合は、アプリケーションの再起動を実施します。

### 3. エスカレーション

一次対応で復旧しない場合は、チームリーダーへ速やかに連絡します。

### 4. 復旧後の対応

1. 復旧したことを監視ダッシュボードで確認します。
2. ユーザーへ復旧完了を連絡します。
3. 障害報告書を作成します。

## 注意事項

- 用語は[用語集](../用語集.md)に従って統一してください。
- 操作スクリーンショットは最新の状態に保ってください。
- 復旧優先で対応し、原因調査は復旧後に実施します。
