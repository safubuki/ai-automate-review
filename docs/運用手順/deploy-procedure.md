# 本番環境へのデプロイ手順

## 目的

本手順は、mainブランチへのマージ後に本番環境へ安全にデプロイするための標準的な流れを定めるものです。

## 前提条件

- デプロイ権限を持つアカウントでGitHubにログインしていること
- 本番環境へのアクセス権限が付与されていること
- 直近のCI/CDパイプラインが成功していること

## 手順

### 1. デプロイ対象の確認

1. GitHubリポジトリの **Actions** タブを開く
2. 最新の `Deploy to Production` ワークフローが `Success` になっていることを確認

### 2. デプロイの実行

1. GitHubの **Environments** から `production` を選択
2. 「Deploy to production」ボタンをクリック
3. 確認ダイアログで `DEPLOY` と入力して実行

### 3. デプロイ後の確認

1. 本番環境のヘルスチェックエンドポイントにアクセス
   ```
   https://example.com/health
   ```
2. レスポンスが `{"status":"healthy"}` であることを確認
3. 主要機能の簡易動作確認を実施（ログイン、トップページ表示、検索）

### 4. 異常検知時の対応

- 5分以内に異常を発見した場合は、**即座にロールバック**を実施
- ロールバック手順は「ロールバック手順書（作成予定）」を参照

## 注意事項

- 金曜日の夕方以降のデプロイは原則禁止
- 大規模な機能リリース時は、事前に運用担当者へSlackで通知すること
- デプロイ後は少なくとも30分は監視を継続
