# ドキュメント貢献ガイド

## 目的

本リポジトリのドキュメントを改善するための貢献方法を定めます。

## 基本的な流れ

1. Issue で改善提案を行う（大規模な変更の場合）
2. 機能ブランチを作成する
3. 変更を加える
4. Pull Request を作成する
5. AIレビュー + 人間レビューを受ける
6. main ブランチへマージ

## 執筆時のチェックリスト

作成・修正時は必ず以下の点を自己確認してください。

- [ ] 用語集に記載されている用語を正しく使用している
- [ ] ページ冒頭に「目的」セクションがある
- [ ] 手順書の場合は「前提条件」「手順」「注意事項」「関連ドキュメント」の4要素を満たしている
- [ ] 見出しはH1を1つだけにし、H2/H3の粒度を統一している
- [ ] 内部リンクは相対パスで記述し、リンク切れがない
- [ ] 画像にはalt textを記載している
- [ ] コードブロックには言語指定をしている
- [ ] ですます調で統一している（手順書の場合）

## Pull Request時のコメント例

```
## 変更概要
- 〇〇手順書の「前提条件」セクションを追記
- 用語の揺れを修正（ユーザー → 統一）

## 確認してほしい点
- 手順の再現性
- トーンの適切性
```

## 禁止事項

- いきなりmainに直接コミットすること
- 用語集を無視した独自表現の使用
- スクリーンショットの未更新放置
