# 用語集

このドキュメント群で使用する用語の定義です。
すべてのマニュアルは本用語集に従って記述してください。

## 基本用語

| 用語 | 定義 | 読み | 備考 |
|------|------|------|------|
| ユーザー | 本サービスを利用するすべての個人または組織 | ゆーざー | 「ユーザ」は使用禁止 |
| 管理者 | 組織内で権限を持つ役割 | かんりしゃ | - |
| 運用担当者 | 日々の運用業務を担当する役割 | うんようたんとうしゃ | - |
| 本番環境 | 実際にサービスを提供している環境 | ほんばんかんきょう | 「プロダクション」は使用可だが、初出時は括弧書きで補足 |
| ステージング環境 | 本番相当の動作確認を行う環境 | すてーじんぐかんきょう | - |

## 技術用語

| 用語 | 定義 | 備考 |
|------|------|------|
| Pull Request | 変更を提案するためのGitHubの機能 | 初出時は「Pull Request (PR)」と表記 |
| デプロイ | アプリケーションや設定を環境に反映する作業 | 「リリース」との違いに注意 |
| ロールバック | 障害発生時に以前の状態に戻す作業 | - |
| インシデント | サービスに影響を与える事象 | 「障害」と混同しない |

## 禁止表現・注意事項

- 「ユーザ」は使用しない → 「ユーザー」を使用
- 「デプロイする」は「デプロイを実施する」または「デプロイする」の両方可だが、文章内で統一
- 「〜と思います」は原則使用しない（特に手順書）
- スクリーンショットは原則として最新の状態に保つ
